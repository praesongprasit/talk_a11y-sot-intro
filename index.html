When?

How early should devs be involved? Depends

What kind of project? Is it a technical solution? Or is it a brochureware site?

Problem statements can be shared early

User research sessions can have devs join in

Data intensive projects can benefit from Dev wireframing. Yes UXers can do it with Axure, but devs don‚Äôt have time to think about data models

How?

  • Ask
  • Align test session with appropriate dev
  • Roster devs
  • Turn them into observers and/or note taker
  • Share findings with the team

A lot of what I taked about means double budget (dev + design) early, which pays off

Some devs, especially BEDs, see no point in this. FEDs are often on board

When companies first try, they make devs sit in every testing session, both user and usability test. Pointless, deminishing return

Data heavy prototype, get devs to sit in first session, fix, and followup

Final synthesised findings give devs idea of how decisions are reached, important to them and not
